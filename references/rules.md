# Prompt 硬规则库 (Hard Rules Library)

> 本文件是 prompt-lint 的确定性规则层，包含 40+ 条可机械检测的规则。
> 每条规则包含：严重级别（critical/warning/info）、正则或关键词检测模式、修复建议。
> 评估器逐条扫描 prompt 文本，命中的规则输出对应级别告警。

---

## 一、SEC — 敏感信息泄露（5 条）

### SEC-001: API Key 格式检测
- **严重级别:** critical
- **检测模式:** `sk-[A-Za-z0-9]{20,}`, `AKIA[A-Z0-9]{16}`, `ghp_[A-Za-z0-9]{36}`, `xoxb-[0-9]+-[A-Za-z0-9]+`, `AIza[A-Za-z0-9_-]{35}`
- **修复建议:** 移除真实 API Key，使用占位符如 `<YOUR_API_KEY>` 或环境变量引用。

### SEC-002: Bearer Token 模式
- **严重级别:** critical
- **检测模式:** `Bearer\s+[A-Za-z0-9\._\-]{20,}`
- **修复建议:** 移除 Bearer Token，改用 `<YOUR_TOKEN>` 占位符。

### SEC-003: 密钥/密码赋值
- **严重级别:** critical
- **检测模式:** `(password|passwd|secret|token|api_key|apikey)\s*[:=]\s*['"]?[^'"\s]{8,}`
- **修复建议:** 移除硬编码密码，使用环境变量或 `<YOUR_PASSWORD>` 占位符。

### SEC-004: 数据库连接字符串
- **严重级别:** critical
- **检测模式:** `(mysql|postgres|postgresql|mongodb|redis|mssql)://[^\s'"]+`
- **修复建议:** 移除连接字符串中的真实凭据，使用 `<DB_HOST>` 等占位符。

### SEC-005: JWT Token
- **严重级别:** critical
- **检测模式:** `eyJ[A-Za-z0-9_-]{10,}\.[A-Za-z0-9_-]{10,}\.[A-Za-z0-9_-]{10,}`
- **修复建议:** 移除 JWT Token，使用 `<YOUR_JWT>` 占位符。

## 二、EFF — Token 效率（6 条）

### EFF-001: 填充词（Filler Phrases）
- **严重级别:** warning
- **检测模式:** `(请帮我|麻烦你|能不能|可不可以|不好意思|拜托了|谢谢|感谢|多谢|辛苦了|劳驾|有劳|麻烦帮忙|麻烦一下)`
- **修复建议:** 直接说任务目标，删除礼貌填充词，可节省 10-30 tokens。

### EFF-002: 冗余表达
- **严重级别:** warning
- **检测模式:** `(实际上|事实上|在某种程度上|从某种意义上说|如果我没记错的话|据我所知|我个人认为|在我看来|我觉得)`
- **修复建议:** 删除冗余修饰语，直接陈述核心意图。

### EFF-003: 过度礼貌
- **严重级别:** info
- **检测模式:** `(非常感谢|万分感谢|太感谢了|太好了|感谢不尽|不胜感激|感激涕零|铭感五内)`
- **修复建议:** 适度礼貌即可，避免过度表达浪费 token。

### EFF-004: 冗长前言（>3 句话才到主题）
- **严重级别:** warning
- **检测模式:** 检测 prompt 前 200 字内是否包含超过 3 个句号/问号/感叹号但不含任务关键词（如"写"、"分析"、"生成"、"翻译"、"总结"等）
- **修复建议:** 将核心任务指令放在前 2 句话，背景信息后置或省略。

### EFF-005: 空泛限定词
- **严重级别:** info
- **检测模式:** `(大概|可能|也许|或许|差不多|基本上|基本上来说|一般来说|通常|一般来说|差不多)`
- **修复建议:** 用具体数字或明确标准替代模糊限定词（如"大概 500 字"改为"500 字"）。

### EFF-006: 重复指令
- **严重级别:** warning
- **检测模式:** 同一语义的指令在 prompt 中出现 2 次以上（如"总结"出现 3 次，"分析"出现 3 次）
- **修复建议:** 合并重复指令，只保留一次明确表达。

## 三、ERR — 错误模式（35 条）

### 任务类（ERR-001 ~ ERR-007）

#### ERR-001: 模糊动词
- **严重级别:** warning
- **检测模式:** `(搞一下|弄一下|处理一下|弄弄|处理处理|搞一搞|弄一弄|看看|看一下|搞个|弄个)`
- **修复建议:** 用明确动词替换（如"写一份"、"分析"、"重构"、"生成"）。

#### ERR-002: 双重任务
- **严重级别:** warning
- **检测模式:** prompt 中同时出现 2 个以上独立任务动词且无分隔（如"写一份文档并生成代码和测试"）
- **修复建议:** 拆分为独立 prompt，每个 prompt 只做一件事。

#### ERR-003: 无成功标准
- **严重级别:** warning
- **检测模式:** prompt 中不含以下关键词之一：`成功标准|验收条件|输出格式|期望结果|示例|通过条件|符合.*要求`
- **修复建议:** 添加明确的成功标准，如"输出需包含 X、Y、Z"。

#### ERR-004: 过度授权
- **严重级别:** info
- **检测模式:** `(随意|随便|你决定|都行|随便你|你看着办|你说了算|自行判断)`
- **修复建议:** 明确约束和偏好，避免让模型自由发挥导致不可控。

#### ERR-005: 情绪化表达
- **严重级别:** info
- **检测模式:** `(生气|愤怒|烦死了|崩溃|无语|受不了|要疯了|气死我了|我裂开了)`
- **修复建议:** 保持客观描述问题，情绪化表达不影响模型输出质量。

#### ERR-006: 任务范围过大
- **严重级别:** warning
- **检测模式:** prompt 中同时包含 5 个以上不同领域/模块的任务（如"前端+后端+数据库+部署+文档+测试"）
- **修复建议:** 将大任务拆分为 3-5 个可执行的子任务，分步完成。

#### ERR-007: 隐式引用
- **严重级别:** warning
- **检测模式:** `(那个|这个|它|那个东西|刚才那个|上面那个|之前那个|这个功能|那个接口)`
- **修复建议:** 明确引用对象的名称和位置（如"src/utils.ts 中的 parseConfig 函数"）。

### 上下文类（ERR-008 ~ ERR-013）

#### ERR-008: 假设已有知识
- **严重级别:** warning
- **检测模式:** `(如你所知|你应该知道|你肯定知道|显而易见|众所周知|不言而喻|不用我说)`
- **修复建议:** 提供完整上下文，不要假设模型知道特定项目信息。

#### ERR-009: 无项目上下文
- **严重级别:** warning
- **检测模式:** prompt 中不含项目名称、技术栈、文件路径等项目相关信息
- **修复建议:** 添加项目背景：项目名称、技术栈、当前进度、关键文件路径。

#### ERR-010: 忘记技术栈
- **严重级别:** warning
- **检测模式:** prompt 中使用了特定框架/语言的术语但未声明技术栈（如提到"组件"但未说 React/Vue）
- **修复建议:** 明确声明使用的技术栈和版本（如"React 18 + TypeScript 5.0"）。

#### ERR-011: 邀请幻觉
- **严重级别:** critical
- **检测模式:** `(随便编|编一个|杜撰|虚构|假设.*存在|如果没有就编)`
- **修复建议:** 要求基于真实信息回答，不确定时明确说"我不知道"。

#### ERR-012: 未定义受众
- **严重级别:** info
- **检测模式:** prompt 中不含受众描述（如"给新手"、"给高级开发者"、"给产品经理"）
- **修复建议:** 明确受众，让模型调整输出深度和用语。

#### ERR-013: 未提及失败情况
- **严重级别:** info
- **检测模式:** prompt 中不含以下关键词之一：`失败|错误|异常|报错|边界条件|失败时|如果出错`
- **修复建议:** 添加失败处理说明，如"如果文件不存在，提示用户上传"。

### 格式类（ERR-014 ~ ERR-019）

#### ERR-014: 无输出格式
- **严重级别:** warning
- **检测模式:** prompt 中不含以下关键词之一：`格式|JSON|Markdown|表格|列表|代码块|YAML|XML|CSV|输出为`
- **修复建议:** 指定期望输出格式（如"以 Markdown 表格输出"、"返回 JSON 格式"）。

#### ERR-015: 隐式长度要求
- **严重级别:** info
- **检测模式:** prompt 中不含以下关键词之一：`字数|字符|长度|页数|段落|行数|简短|详细|简洁|至少|最多|不超过`
- **修复建议:** 明确长度要求（如"500 字以内"、"3-5 段"、"不超过 100 行"）。

#### ERR-016: 无角色设定
- **严重级别:** info
- **检测模式:** prompt 中不含以下关键词之一：`作为|你是|扮演|身份|角色|专家|工程师|设计师|分析师`
- **修复建议:** 添加角色设定以提升输出质量（如"你是一位资深 Python 工程师"）。

#### ERR-017: 模糊美学要求
- **严重级别:** info
- **检测模式:** `(好看|漂亮|美观|高级感|设计感|有质感|大气|简洁一点|花哨一点)`
- **修复建议:** 用具体设计指标替代（如"使用深色主题，主色调 #1a1a2e"）。

#### ERR-018: 无负面提示
- **严重级别:** info
- **检测模式:** prompt 中不含以下关键词之一：`不要|避免|禁止|别|不能|不许|切勿`
- **修复建议:** 添加负面约束以减少不需要的输出（如"不要使用第三方库"）。

#### ERR-019: 错误模板
- **严重级别:** warning
- **检测模式:** prompt 中包含其他 AI 工具的特征标记（如"### System:"、"Human:"、"Assistant:"、"```python```" 但非代码块）
- **修复建议:** 移除其他工具的模板标记，保持 prompt 干净。

### 范围类（ERR-020 ~ ERR-025）

#### ERR-020: 无边界声明
- **严重级别:** warning
- **检测模式:** prompt 中不含以下关键词之一：`范围|边界|限于|仅限|只做|不涉及|不包括|排除|除了`
- **修复建议:** 明确任务边界（如"仅修改 src/auth 模块，不涉及其他模块"）。

#### ERR-021: 无技术约束
- **严重级别:** warning
- **检测模式:** prompt 中不含以下关键词之一：`使用|采用|基于|依赖|不使用|禁止使用|必须用|优先用`
- **修复建议:** 明确技术约束（如"使用原生 CSS，不使用 Tailwind"）。

#### ERR-022: 无停止条件
- **严重级别:** warning
- **检测模式:** prompt 中不含以下关键词之一：`完成条件|停止条件|什么时候停|当.*时结束|循环.*次|最多.*次`
- **修复建议:** 添加明确停止条件（如"当测试通过时停止"、"最多迭代 3 次"）。

#### ERR-023: 无文件路径
- **严重级别:** warning
- **检测模式:** prompt 中不含以下路径模式：`/.*\.|src/|lib/|pages/|components/|tests/|\.ts|\.js|\.py|\.go`
- **修复建议:** 指定具体文件路径（如"修改 src/components/Button.tsx"）。

#### ERR-024: 无禁止清单
- **严重级别:** info
- **检测模式:** prompt 中不含以下关键词之一：`不要改|不要动|不要碰|禁止修改|保持不变|原样保留|保留.*不动`
- **修复建议:** 列出不应修改的部分（如"不要修改 tests/ 目录下的文件"）。

#### ERR-025: 无人工审核环节
- **严重级别:** info
- **检测模式:** prompt 中不含以下关键词之一：`审核|检查|确认|验证|review|检查一遍|确认一下`
- **修复建议:** 添加人工审核步骤（如"修改后请我确认再提交"）。

### 推理类（ERR-026 ~ ERR-030）

#### ERR-026: 逻辑任务未要求 CoT
- **严重级别:** warning
- **检测模式:** prompt 中含逻辑推理任务（如"分析原因"、"推理"、"判断"、"比较"、"评估"）但不含 CoT 指令（如"逐步思考"、"chain of thought"、"推理过程"、"解释你的思路"）
- **修复建议:** 添加 CoT 指令以提升推理质量（如"请逐步分析原因"）。

#### ERR-027: 推理模型上用了 CoT
- **严重级别:** info
- **检测模式:** prompt 中含"逐步思考"/"chain of thought"等 CoT 指令，但模型为 o1/o3/o4-mini 等推理模型
- **修复建议:** 推理模型自带 CoT，无需额外指令，删除 CoT 相关提示。

#### ERR-028: 无自检步骤
- **严重级别:** info
- **检测模式:** prompt 中不含以下关键词之一：`自检|自查|检查.*是否|验证.*是否|确认.*是否|review|校验`
- **修复建议:** 添加自检步骤（如"生成后检查代码是否能通过 lint"）。

#### ERR-029: 跨会话记忆假设
- **严重级别:** warning
- **检测模式:** `(上次|之前|前面|刚才|之前提到的|上次讨论的|我们之前|你还记得)`
- **修复建议:** 提供完整上下文，不要假设模型记得之前的对话。

#### ERR-030: 与先前决策矛盾
- **严重级别:** info
- **检测模式:** prompt 中含"但是"、"不过"、"然而"等转折词且后接与前文矛盾的指令
- **修复建议:** 检查并统一指令，避免前后矛盾。

### Agent 类（ERR-031 ~ ERR-035）

#### ERR-031: 无起始状态
- **严重级别:** warning
- **检测模式:** prompt 中不含以下关键词之一：`当前|现有|现状|目前|现在|起始|初始|从.*开始|基于现有`
- **修复建议:** 描述当前状态（如"当前项目使用 React 18，已有登录功能"）。

#### ERR-032: 无目标状态
- **严重级别:** warning
- **检测模式:** prompt 中不含以下关键词之一：`目标|期望|最终|完成后|结果是|应该是|需要变成`
- **修复建议:** 描述期望的最终状态（如"完成后应支持用户注册和登录"）。

#### ERR-033: 静默 Agent
- **严重级别:** info
- **检测模式:** prompt 中不含以下关键词之一：`进度|状态|报告|反馈|输出.*过程|记录.*日志|打印.*信息`
- **修复建议:** 要求 Agent 输出进度信息（如"每完成一个步骤输出进度"）。

#### ERR-034: 无文件系统锁
- **严重级别:** info
- **检测模式:** prompt 中含多步骤文件操作但不含以下关键词之一：`锁|锁定|原子|事务|回滚|备份|撤销`
- **修复建议:** 添加文件操作安全性说明（如"修改前先备份原文件"）。

#### ERR-035: 无人工审核节点
- **严重级别:** info
- **检测模式:** prompt 中含多步骤操作但不含以下关键词之一：`确认|审核|approve|review|检查.*再.*|等我确认`
- **修复建议:** 在关键步骤后添加人工审核节点（如"生成配置后等我确认再执行"）。

### FMT-004: 无角色设定

## 使用说明

1. **评估流程：** 将待评估 prompt 逐条匹配本规则库，命中任一规则即记录告警。
2. **严重级别含义：**
   - `critical`：必须修复，存在安全风险或严重质量问题。
   - `warning`：建议修复，影响输出质量或效率。
   - `info`：可选优化，提升 prompt 专业性。
3. **检测方式：** 正则匹配 + 关键词扫描，支持中英文混合检测。
4. **输出格式：** 每条命中的规则输出 `RULE-ID: 规则名 | 严重级别 | 修复建议`。
